# Section Strategy

Choose resume/application sections before selecting events.

## Standard Section Registry

Prefer conventional section IDs:

- `summary`
- `skills`
- `work_experience`
- `internship_experience`
- `projects`
- `research`
- `education`
- `publications`
- `awards`
- `certifications`
- `portfolio`

Custom sections are allowed only when the target needs them. A custom section must include `title`, `purpose`, `source_events`, and `ats_risk`.

## Selection Rules

- Target/JD decides the section strategy.
- Sections define evidence needs.
- Timeline events are selected after sections exist.
- One-page resumes usually use 4-5 main sections.
- Two-page resumes can use 5-7 sections.
- ATS mode uses conventional headings.
- Visual/networking mode may use richer titles, but avoid gimmicky titles.

## Resume Plan Must Show

- target positioning
- selected sections in order
- purpose of each section
- selected events per section
- omitted but relevant events and why
- gaps and risks
- planned page count
- planned design mode
