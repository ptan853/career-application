# Target Research

Build a structured target profile before writing application text.

## Inputs

- JD text or URL
- company and role
- target country/region
- application channel: ATS, recruiter, referral, networking, portfolio
- requested artifact: resume, cover letter, interview prep, portfolio summary

## Research Questions

- What does this target hire for?
- Which evidence is likely to pass the initial screen?
- Which evidence would make the candidate stand out?
- What keywords and domain language should be used naturally?
- What risks or gaps should be made explicit instead of hidden?

## Output

Create or update `target.json` with:

- company/product/domain context
- role responsibilities
- must-have and nice-to-have skills
- seniority signals
- hiring priorities
- keywords
- risks
- source URLs and access dates

If research is based only on user-provided JD text, mark sources as `user_provided`.
